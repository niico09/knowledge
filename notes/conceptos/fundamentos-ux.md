# Fundamentos y Principios Core — UX/UI

## Proceso DCU — Diseño Centrado en el Usuario

5 fases **iterativas**, no lineales:

| Fase | Objetivo |
|------|----------|
| Empatizar | Conocer al usuario real |
| Definir | Convertir hallazgos en problemas accionables |
| Idear | Generar soluciones antes de filtrar |
| Prototipar | Validar antes de invertir en desarrollo |
| Testear | Validar con usuarios reales |

## Las 10 Heurísticas de Nielsen

1. **Visibilidad del estado** — Siempre informar qué está pasando
2. **Correspondencia con el mundo real** — Hablar el idioma del usuario
3. **Control y libertad** — Siempre ofrecer salida de emergencia
4. **Consistencia y estándares** — "Guardar" siempre igual
5. **Prevención de errores** — Deshabilitar hasta que sea válido
6. **Reconocimiento antes que recuerdo** — Autocompletar, historial
7. **Flexibilidad y eficiencia** — Shortcuts para expertos
8. **Diseño minimalista** — Eliminar lo que no agrega valor
9. **Recuperación de errores** — [Qué] + [Por qué] + [Cómo resolver]
10. **Ayuda y documentación** — Tooltips contextuales

## Leyes Cognitivas

| Ley | Principio | Aplicación |
|-----|-----------|------------|
| Hick | Más opciones = más tiempo | Máx 5-7 ítems en nav |
| Fitts | Grande y cerca = fácil | Botones ≥ 44×44px |
| Miller | 7±2 chunks en memoria | Agrupar contenido |
| Jakob | Convenciones conocidas | No reinventar |

## Métricas HEART (Google)

| Dimensión | Qué mide | Ejemplo |
|-----------|----------|---------|
| Happiness | Satisfacción | NPS, SUS |
| Engagement | Involcramiento | Sesiones/semana |
| Adoption | Nuevos usuarios | % activación |
| Retention | Usuarios que regresan | D7, D30 |
| Task Success | Eficiencia | Task Completion Rate |

**SUS:** ≥ 68 = bueno, ≥ 80 = excelente.
