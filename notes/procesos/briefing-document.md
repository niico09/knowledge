# NotebookLM Briefing Document

Prompt para generar informe técnico estructurado a partir de un capítulo.

## Estructura del output

1. **Executive Summary** — Paradigm shift + problema core
2. **Detailed Analysis** — 3-5 subsections con tablas/listas
3. **Conclusions & Trade-offs** — Tabla de decisiones
4. **Open Questions** — Preguntas sin resolver

## Flujo de recuperación activa

1. Leer Briefing en pantalla (una vez)
2. Cerrar pantalla
3. En papel: reconstruir trade-offs + diagrama + pregunta abierta
4. Comparar con Briefing
5. Llevar gaps al Learning Engine
