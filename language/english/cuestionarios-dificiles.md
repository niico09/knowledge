## CUESTIONARIO 1: Strunk & White — Extended (30 preguntas)

```
Generá un quiz de 30 preguntas de alta dificultad basado en "The Elements of Style" de Strunk & White.
Nivel: B2-C1. Escenario: Sos senior tech writer en una empresa de software.

## PARTE A — Opción múltiple compleja (10 preguntas)
Cada pregunta tiene 5 opciones. Elegí la MEJOR versión.

1. ¿Cuál es la versión más concisa y clara?
   a) "The system which processes the authentication requests is critical"
   b) "The system that processes authentication requests is critical"
   c) "The system, which processes the authentication requests, is critical"
   d) "The critical system processes authentication requests"
   e) "The system that is processing authentication requests is the critical one"

2. ¿Dónde está el error en esta oración?
   "All of the team members was informed about the deployment schedule."
   a) subject-verb agreement (was/were)
   b) word order
   c) article usage
   d) tense consistency
   e) preposition choice

3. ¿Cuál es la forma correcta?
   a) "Neither the manager nor the developers were aware"
   b) "Neither the manager nor the developers was aware"
   c) "Neither the managers nor the developer was aware"
   d) "Either the manager or the developers was aware"
   e) "Neither the manager or the developers were aware"

4. Seleccioná la oración que viola la regla de "omit needless words":
   a) "The function returns the calculated value"
   b) "The function returns a value that has been calculated"
   c) "The function returns the calculated result"
   d) "The function returns what was calculated"
   e) "The calculated value is returned by the function"

5. ¿Cuál es el error principal?
   "Being that the API is down, we are unable to process requests."
   a) Wordiness ("Being that" → "Since")
   b) Passive voice
   c) Subject omission
   d) Verb tense
   e) Article error

6. ¿Qué opción corrige correctamente el wordiness sin perder claridad?
   a) "Due to the fact that the API is down, we are unable to process requests"
   b) "Because the API is down, we cannot process requests"
   c) "Owing to the API being down, we are unable to process requests"
   d) "Since the API is down, we are not able to process requests"
   e) "The API being down means we are unable to process requests"

7. ¿Cuál es la regla violada en "The reason is because..."?
   a) Wordiness
   b) Redundant conjunction
   c) Subject-verb disagreement
   d) Passive voice
   e) Wrong idiom

8. ¿Cuál es la forma más clara de escribir "At this point in time"?
   a) "Currently"
   b) "Right now"
   c) "At this time"
   d) "Presently"
   e) "At the present time"

9. ¿Cuál de estas oraciones usa voz activa correctamente?
   a) "The bug was fixed by Sarah in the authentication module"
   b) "The authentication module was fixed by Sarah for the bug"
   c) "Sarah fixed the bug in the authentication module"
   d) "The bug was fixed in the authentication module by Sarah"
   e) "The fixing of the bug was done by Sarah in the module"

10. ¿Cuál es la mejor estructura para esta oración?
    "The meeting is at 3pm, and also there will be a demo of the new feature."
    a) "The meeting is at 3pm; additionally, there will be a demo of the new feature"
    b) "At 3pm, the meeting will include a demo of the new feature"
    c) "The meeting starts at 3pm, and also a demo of the new feature will be presented"
    d) "There will be a meeting at 3pm, and a demo of the new feature will also happen"
    e) "At 3pm we will have a meeting, and additionally there will be a demo"

## PARTE B — Corrección avanzada (10 preguntas)
Corrigí cada texto aplicando TODAS las reglas de Strunk & White.

11. "In order to be able to understand the codebase, it is necessary to read the documentation first."

12. "The reason why the deployment failed is because the configuration was not correct."

13. "This issue has been looked into by the team and they are working on it."

14. "Due to the fact that there were many bugs, the release was postponed."

15. "The function which processes user input, validates it, and then stores the result."

16. "In regards to your email, I wanted to let you know that we have reviewed it."

17. "The API as well as the database needed to be restarted in order to fix the issue."

18. "All of the information which you need can be found in the documentation that is provided."

19. "It is recommended that you should backup your data before proceeding with the update."

20. "The system was developed by using the latest framework and was deployed by using CI/CD."

## PARTE C — Producción escrita (5 preguntas)
Escribí un párrafo técnico para cada scenario, aplicando STRICTAMENTE las reglas de Strunk & White.

21. Explicá por qué el deployment falló. Usá máximo 50 palabras. Sin wordiness.

22. Describí el proceso de authentication de un sistema. Voz activa. Máximo 60 palabras.

23. Redactá un paragrapho de changelog para version 2.0. Sé conciso. Máximo 40 palabras.

24. Explicá "drill down" como si fuera documentación. Eliminate TODAS las words innecesarias.

25. Escribí una explicación del bug fix a un stakeholder NO técnico. Claridad máxima. Máximo 80 palabras.

## PARTE D — Identificación de reglas (5 preguntas)
Para cada oración, identificá qué regla de Strunk & White se viola.

26. "The code review is something that I think is important."
27. "This is a problem which is difficult to solve."
28. "In my opinion, I think that the approach is correct."
29. "The database, which contains user data, is critical."
30. "He delivered the presentation in a manner which was unclear and confusing."

## CUESTIONARIO 3: Idioms Técnicos — Extended (30 preguntas)

```
