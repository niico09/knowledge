# Prompts NotebookLM — English B2 Tech Path

Colección de prompts para cada fase del sistema de estudio.

---

## FASE 1: BRIEFING DOCUMENT

### Prompt Genérico de Briefing

```
Genera un Briefing Document estructurado con:
1. Resumen ejecutivo (5-7 oraciones)
2. Conceptos clave (5-10 puntos)
3. Términos técnicos importantes (glosario)
4. Ideas contrapuestas o debates en el texto
5. Preguntas guía para profundización
6. Recursos adicionales sugeridos

Lenguaje: Claro, en inglés, apropiado para nivel B2-C1.
```

### Briefing: The Elements of Style

```
Crea un Briefing Document para los capítulos 1-3 de "The Elements of Style" de Strunk & White.
Focus en:
- Las 8 reglas de uso más importantes
- La diferencia entre "which" y "that"
- Cómo identificar oraciones wordy
- Casos donde el estilo informal es apropiado vs formal
Incluye 5 ejemplos de aplicación técnica para cada concepto.
```

### Briefing: English Collocations in Use (Unit 1-10)

```
Genera un Briefing Document para las Units 1-10 de "English Collocations in Use" (McCarthy & O'Dell).
Para cada unit:
- 5 colocaciones más útiles del ámbito tecnológico/profesional
- Patrones gramaticales (verb + noun, adj + noun, etc.)
- Errores comunes que hacen estudiantes de nivel B2
- 2 ejemplos de uso en contexto técnico
Lenguaje: Examples en inglés, explicaciones en español para claridad.
```

### Briefing: Technical Communication (Markel)

```
Crea un Briefing Document del Capítulo 1-4 de "Technical Communication" (Mike Markel).
Estructura:
- Principios de comunicación técnica efectiva
- Análisis de audiencia: cómo adaptar el mensaje
- Diseño de documentos: estructura visual y jerarquía
- Ética en comunicación técnica
Incluye: 3 templates listos para usar (email técnico, memo, breve report).
```

### Briefing: Idioms for Technical Professionals

```
Genera un Briefing Document con 30 idioms/expressions esenciales para profesionales tech.
Para cada idiom:
- Significado literal y figurado
- Ejemplo de uso en contexto técnico (reunión, code review, bug discussion)
- Nivel de formalidad (formal/neutral/informal)
- Variantes si existen
Agrupados por categoría: troubleshooting, meetings, presentations, disagreements.
```

## FASE 2: GUÍA DE RECONSTRUCCIÓN

### Guía de Reconstrucción Genérica

```
Basado en el material cargado, genera una "Guía de Reconstrucción" para llevar a papel.
Estructura:
1. Mapa conceptual del tema (formato ASCII o texto)
2. 10-15 puntos clave que debés poder explicar sin mirar el libro
3. 5 analogías o metáforas para recordar conceptos difíciles
4. Lista de "cosas que voy a olvidar" ( traps comunes)
5. Checkpoint de auto-evaluación (5 preguntas de dificultad media)
Tiempo estimado: 20-25 minutos de estudio activo.
```

### Guía: Writing for Technology (Reglas de Strunk & White)

```
Genera una Guía de Reconstrucción enfocada en las reglas de escritura técnica.
Debés poder explicar SIN mirar el libro:
1. Por qué "use active voice" es casi siempre mejor en docs técnicos
2. Cómo identificar y eliminar words innecesarias
3. Cuándo usar listas vs oraciones largas
4. Diferencia entre "which" y "that" y por qué importa
5. Cómo estructurar un párrafo técnico: orden lógico
Incluye: 1 paragraph técnico "antes" y "después" de aplicar las reglas.
```

### Guía: Colocaciones Tech

```
Crea una Guía de Reconstrucción para dominarlas 50 colocaciones más útiles de technology & business.
Formato: flashcards mentales para practicar en papel
Ejemplo de formato:
┌─────────────────────────────────────┐
│ make [X]  ❌  do [X]                 │
│ ✓ make a decision                   │
│ ✓ make progress                      │
│ ✓ make an exception                  │
│ X do a decision (WRONG)              │
│ X do progress (WRONG)                │
└─────────────────────────────────────┘
Incluye: 50 tarjetas en formato texto, listo para pasar a Anki o Quizlet.
```

## FASE 3: TARJETAS DIDÁCTICAS

### Prompt Genérico de Tarjetas

```
Genera 20 flashcards (formato Anki/Quizlet) con:
Pregunta al frente (inglés), Respuesta atrás (inglés con ejemplo).
Focus en:
- Preguntas que revelen gaps de conocimiento
- Ejemplos del ámbito tecnológico/profesional
- Patrones, no casos aislados
- Excepciones a reglas generales

Formato:
---
Q: [pregunta]
A: [respuesta] | e.g. "example sentence in context"
---
```

### Tarjetas: Strunk & White Rules

```
Genera 30 flashcards para dominar "The Elements of Style":
Nivel: B2-C1. Contexto: escritura técnica en inglés.
Incluye:
- 10 reglas de uso (which/that, active voice, etc.)
- 10 colocaciones formales vs informales
- 10 corrections (wordy → concise)

Formato Anki:
Q: Which is correct: "the system that" or "the system which"?
A: "the system that" (restrictive clause) | "The module that handles auth is critical"

Q: Rewrite MORE CONCISE: "Due to the fact that the API was down, we were unable to complete the task."
A: "Because the API was down, we couldn't complete the task."

Q: ACTIVE or PASSIVE? "The bug was fixed by the developer"
A: PASSIVE → Better: "The developer fixed the bug"
```

### Tarjetas: Colocaciones Tech

```
Genera 50 flashcards de colocaciones para tech professionals.
Categorías:
- 15: verb + noun (make/do/build/write/run)
- 15: tech actions (deploy, debug, integrate, optimize)
- 10: meeting phrases (raise, follow up, weigh up)
- 10: presentation phrases (walk through, hand over, sum up)

Formato:
Q: What collocation means "investigar en detalle"?
A: drill down into | e.g. "Let's drill down into the root cause"

Q: Which is correct: "make a decision" or "do a decision"?
A: make a decision (not "do")
```

### Tarjetas: Idioms Técnicos

```
Genera 25 flashcards con idioms para conversaciones técnicas.
Incluye:
- Significado + ejemplo técnico
- Nivel de formalidad
- Situaciones para usar

Q: What does "hit a snag" mean? Give a tech example.
A: Encontrar un obstáculo inesperado | "We hit a snag during deployment when the CI pipeline failed."

Q: Complete: "Let me ___ this concept before we move on."
A: drill down into | "Let me drill down into this concept"

Q: What's the difference between "get something off the ground" and "launch"?
A: Same meaning, but "launch" is more formal. "Get off the ground" is neutral/informal.
```

## FASE 4: CUESTIONARIO

### Cuestionario de Diagnóstico Genérico

```
Genera un cuestionario de 15 preguntas para validar comprensión.
Tipos de preguntas:
- 5 de opción múltiple (conceptos)
- 5 de completar/cloze (vocabulario, collocations)
- 5 de corrección (identificar error y corregir)

Nivel: B2-C1
Focus: detectar gaps residuales, no lo que ya sabés.
```

### Cuestionario: Strunk & White Aplicado

```
Crea un quiz de 20 preguntas para validar aplicación de Strunk & White.
escenario: Eres tech writer en una empresa de software.

PARTE A - Corrección (10 preguntas):
Corrigé el texto técnico. Aplicá las reglas de Strunk & White.

PARTE B - Opción múltiple (5 preguntas):
¿Cuál es la versión más clara?

PARTE C - Producción (5 preguntas):
Escribí un párrafo técnico aplicando todas las reglas aprendidas.

Criterio de aprobación: 80% (16/20)
```

### Cuestionario: Collocations Check

```
Genera quiz de 25 preguntas para validar colocaciones.
Formato:
- 10: Elegir collocation correcta (opción múltiple)
- 10: Completar oración con collocation apropiada
- 5: Traducir frase al inglés usando collocations correctas

Tematicas: technology, meetings, problem-solving, writing, presenting.
```

## FASE 5: AUDIO OVERVIEW

### Prompt para Audio Overview

```
Crea un Audio Overview (podcast de NotebookLM) enfocado en:
"Cómo escribir documentación técnica clara en inglés"

Debe cubrir:
1. Por qué la claridad importa en docs técnicos (1 min)
2. Las 3 reglas de Strunk & White más importantes (2 min)
3. Colocaciones clave para documentación (2 min)
4. Errores comunes en documentación técnica (2 min)
5. Tips para practicar (1 min)

Total: ~8-10 minutos. Estilo: conversación profesional pero accesible.
```

### Audio Overview: Conversation Phrases

```
Genera Audio Overview para practicar conversación técnica.
Título: "How to Discuss Technical Problems in English"

Contenido:
1. Opening a technical discussion (1 min)
2. Explaining a problem clearly (2 min)
3. Asking for help or clarification (2 min)
4. Giving updates on progress (1.5 min)
5. Ending a technical conversation (1 min)

Incluye: 10 frases esenciales con ejemplo de uso.
```

## FASE 6: DEEP RESEARCH (Opcional)

### Deep Research: Tech Writing Trends

```
Realizá una búsqueda profunda sobre:
"Best practices en documentación técnica para productos de software en 2024-2025"

Debe incluir:
- Frameworks de docs (Diátaxis, etc.)
- Herramientas actuales (GitBook, Mintlify, Docusaurus)
- Tendencias en API documentation
- Ejemplos de excelente documentación (casos de estudio)
- Recomendaciones concretas para mejorar docs propios

Formato: Reporte de 500-800 palabras con links a recursos.
```
